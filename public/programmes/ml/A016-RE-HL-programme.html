Backend Headless avec WordPress et Frontend React

Public concerné :

35 heures (5 jours)
Modalité :
Présentiel ou distanciel

Objectifs pédagogiques

À la fin de cette formation, les participants seront capables de :

  • Configurer et exposer WordPress comme API headless
  • Développer une application frontend moderne avec React
  • Connecter une application React à l'API REST de WordPress
  • Implémenter des fonctionnalités avancées dans une architecture découplée
  • Déployer et maintenir une architecture headless sécurisée et performante

Programme détaillé

JOUR 1 : Fondamentaux des architectures headless et configuration de WordPress

Session 1 : Introduction aux architectures headless

  • Comprendre les avantages et inconvénients des architectures headless
  • Explorer les cas d'usage adaptés aux architectures découplées
  • Présentation de l'écosystème WordPress et React
  • Mise en place de l'environnement de développement

Session 2 : Configuration de WordPress en mode headless

  • Installation et configuration d'un WordPress optimisé pour API
  • Présentation de l'API REST WordPress
  • Configuration des Custom Post Types et taxonomies
  • Exercice pratique : Explorer et tester les endpoints de l'API WordPress
JOUR 2 : Bases de React et création du frontend

Session 1 : Introduction à React

  • Comprendre les fondamentaux de React (composants, props, state)
  • Mise en place d'un projet React avec Create React App ou Next.js
  • Routing et navigation avec React Router
  • Travail avec JSX et les hooks de React

Session 2 : Architecture de l'application frontend

  • Structurer son application React
  • Gestion d'état avec Context API ou Redux
  • Création de composants réutilisables
  • Exercice pratique : Créer la structure de base de l'application React
JOUR 3 : Connecter React à l'API WordPress

Session 1 : Requêtes API et gestion de données

  • Utilisation de Fetch API ou Axios pour les requêtes HTTP
  • Requêtes à l'API WordPress et gestion des réponses
  • Traitement des erreurs et gestion des états de chargement
  • Mise en cache des données et optimisation des performances

Session 2 : Implémentation des fonctionnalités principales

  • Création de la page d'accueil et listing des articles
  • Affichage des articles, pages et médias depuis l'API
  • Implémentation de la recherche et du filtrage
  • Exercice pratique : Développer un affichage dynamique du contenu WordPress
JOUR 4 : Fonctionnalités avancées et authentification

Session 1 : Authentification et gestion utilisateur

  • Implémentation de JWT avec WordPress
  • Création d'un système d'authentification sécurisé
  • Gestion des droits utilisateurs
  • Sécurisation des endpoints et protection des données

Session 2 : Fonctionnalités interactives

  • Mise en place de formulaires dynamiques avec React
  • Création et modification de contenu via l'API
  • Implémentation de commentaires et d'interactions utilisateur
  • Exercice pratique : Développer un système de commentaires avec authentification
JOUR 5 : Déploiement et optimisation

Session 1 : Optimisation des performances

  • Techniques d'optimisation pour React (Code splitting, lazy loading)
  • SEO pour les applications React
  • Server-Side Rendering (SSR) avec Next.js
  • Mise en cache avancée et génération de sites statiques

Session 2 : Déploiement et maintenance

  • Préparation au déploiement (build, environnements)
  • Déploiement sur différentes plateformes (Vercel, Netlify)
  • Stratégies de CI/CD pour WordPress et React
  • Exercice final : Déploiement complet d'une application headless WordPress + React
